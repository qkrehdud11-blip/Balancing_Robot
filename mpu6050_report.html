1 MPU6050 개요 및 주요 사양

MPU6050은 InvenSense(현 TDK)에서 제조한 6축 IMU(Inertial Measurement Unit) 센서입니다. 3축 가속도계와 3축 자이로스코프를 하나의 패키지에 통합하고, I²C 인터페이스로 마이크로컨트롤러/FPGA와 통신합니다.

2 감도(Sensitivity)와 16384의 근거

MPU6050의 가속도계는 16비트 부호있는 정수(signed 16-bit)로 값을 출력합니다. 이 숫자가 실제 물리량(g)으로 변환되는 비율을 감도(Sensitivity Scale Factor)라고 합니다.

계산 원리

16-bit signed 정수 범위 : -32768 ~ +32767
양의 최대값 = 2^15 = 32768

±2g 레인지 선택 시:
Sensitivity = 32768 ÷ 2 (g) = 16384 LSB/g

→ 1g(중력가속도)가 인가되면 ADC 출력 = 16384
// LSB = Least Significant Bit, 즉 최솟값 1단위

레인지별 감도 비교

가속도 레인지SensitivityACCEL_CONFIG
±2g (기본값) 16384 LSB/g AFS_SEL = 0b00
±4g 8192 LSB/g AFS_SEL = 0b01
±8g 4096 LSB/g AFS_SEL = 0b10
±16g 2048 LSB/g AFS_SEL = 0b11
자이로 레인지SensitivityGYRO_CONFIG
±250°/s (기본값) 131 LSB/°/s FS_SEL = 0b00
±500°/s 65.5 LSB/°/s FS_SEL = 0b01

결론: 이 프로젝트는 ACCEL_CONFIG/GYRO_CONFIG를 설정하지 않아 기본값 사용 중.
가속도 ±2g → Sensitivity = 16384 LSB/g
자이로 ±250°/s → Sensitivity = 131 LSB/(°/s)

실제값 변환 공식

실제 가속도 (g) = ADC값 ÷ 16384
실제 각속도 (°/s) = ADC값 ÷ 131

예시: AZ = 16120 → 16120 ÷ 16384 ≈ 0.984g (평지 기준 약 1g, 정상)
예시: GX = 131 → 131 ÷ 131 ≈ 1.0°/s

3 가속도계 데이터 해석

가속도계는 중력(g)을 포함한 선형 가속도를 측정합니다. 정지 상태에서도 중력(1g)이 측정되므로, 이를 기준으로 자세를 판단할 수 있습니다.

평지 정치 시 이상값

상태AXAYAZ설명
평지 정치 (이상) 0 0 +16384 중력이 Z축 방향으로만 작용
X축으로 5° 기울음 ±1428 ~0 ~16320 sin(5°)×16384 ≈ 1428
X축으로 90° 기울음 ±16384 ~0 ~0 중력이 X축 방향으로 작용
완전히 뒤집힘 ~0 ~0 -16384 중력이 -Z 방향
자유낙하 0 0 0 무중력 상태 (모든 축 0)

기울기 각도 계산

θ_x = arctan(AX / AZ) × (180 / π)
θ_y = arctan(AY / AZ) × (180 / π)

예시: AX=1471, AZ=16120
θ_x = arctan(1471 / 16120) = arctan(0.0912) ≈ 5.2°
// 가속도계만으로 계산한 각도 → 노이즈에 민감하므로 자이로와 상호 보완 필요

4 자이로스코프 데이터 해석

자이로스코프는 각속도(°/s, degree per second)를 측정합니다. 정지 상태에서는 0에 가까운 값이 나와야 하며, 회전 중에는 회전 속도에 비례한 값이 출력됩니다.

항목가속도계자이로스코프
측정 대상 선형 가속도 (중력 포함) 각속도 (회전 속도)
정지 시 AZ ≈ 16384 (중력), AX/AY ≈ 0 GX/GY/GZ ≈ 0
장점 절대 각도 계산 가능 (중력 기준) 빠른 응답, 단기 정확도 높음
단점 진동·충격에 민감, 노이즈 큼 드리프트(시간 지남에 따라 오차 누적)
균형 로봇 사용 기준 각도 보정용 실시간 각속도 제어용
균형 로봇 제어 전략: 가속도계 각도(절대 기준) + 자이로 각속도(빠른 반응)를 상보 필터(Complementary Filter) 또는 칼만 필터로 융합하여 안정적인 각도 추정이 필요합니다.

5 I²C 통신 및 레지스터 맵

MPU6050은 I²C 슬레이브로 동작합니다. AD0 핀 레벨로 주소가 결정되며, 이 프로젝트에서는 AD0=GND(0x68)를 사용합니다.

I²C 주소

7-bit 주소: 0x68 (AD0=GND) / 0x69 (AD0=VCC)
쓰기 주소 (8-bit): 0xD0 → { 0x68, 1'b0 }
읽기 주소 (8-bit): 0xD1 → { 0x68, 1'b1 }

주요 레지스터 맵

주소레지스터명기본값설명
0x19SMPLRT_DIV 0x00 샘플링 레이트 분주기 (기본: 8kHz÷1 = 8kHz)
0x1ACONFIG 0x00 DLPF(디지털 저역 통과 필터) 설정
0x1BGYRO_CONFIG 0x00 자이로 레인지: 0x00=±250°/s
0x1CACCEL_CONFIG 0x00 가속도 레인지: 0x00=±2g
0x3BACCEL_XOUT_H - 가속도 X축 상위 바이트 (burst read 시작점)
0x3CACCEL_XOUT_L -가속도 X축 하위 바이트
0x3D~0x40ACCEL_Y/Z -가속도 Y, Z축 (각 2바이트)
0x41~0x42TEMP_OUT -온도 센서 (이 프로젝트 미사용)
0x43~0x48GYRO_X/Y/Z -자이로 X, Y, Z축 (각 2바이트)
0x6B PWR_MGMT_1 0x40 슬립 모드 제어. 기본값 0x40=슬립 ON → 0x00으로 초기화 필요
0x75WHO_AM_I 0x68 칩 ID 확인용 (항상 0x68 반환)

Burst Read 순서 (0x3B~0x48, 14바이트)

Byte 0: ACCEL_XOUT_H │ Byte 1: ACCEL_XOUT_L → AX = {Byte0, Byte1}
Byte 2: ACCEL_YOUT_H │ Byte 3: ACCEL_YOUT_L → AY = {Byte2, Byte3}
Byte 4: ACCEL_ZOUT_H │ Byte 5: ACCEL_ZOUT_L → AZ = {Byte4, Byte5}
Byte 6: TEMP_OUT_H │ Byte 7: TEMP_OUT_L → (skip)
Byte 8: GYRO_XOUT_H │ Byte 9: GYRO_XOUT_L → GX = {Byte8, Byte9}
Byte 10: GYRO_YOUT_H │ Byte 11: GYRO_YOUT_L → GY = {Byte10,Byte11}
Byte 12: GYRO_ZOUT_H │ Byte 13: GYRO_ZOUT_L → GZ = {Byte12,Byte13}

6 초기화 시퀀스

MPU6050은 기본적으로 슬립(Sleep) 모드로 부팅됩니다. 반드시 PWR_MGMT_1 레지스터를 0x00으로 설정해 슬립을 해제해야 데이터가 출력됩니다.

최소 초기화 (현재 프로젝트)

① 전원 인가 후 50ms 이상 대기
② I²C Write: 0x6B (PWR_MGMT_1) ← 0x00 // 슬립 해제, 내부 8MHz 클럭 사용
③ 이후 0x3B부터 14바이트 Burst Read 반복

권장 추가 초기화 (선택)

순서레지스터값효과
1 0x6B 0x00 슬립 해제 (필수)
2 0x1A 0x03 DLPF 활성화 (44Hz 차단, 노이즈 감소)
3 0x19 0x04 샘플레이트 200Hz (DLPF ON 시 1kHz÷(4+1))
4 0x1B 0x08 자이로 레인지 ±500°/s (균형 로봇에 더 적합)
⚠ DLPF 추천: 균형 로봇은 모터 진동이 심합니다. DLPF(CONFIG=0x03, 44Hz 차단주파수)를 설정하면 고주파 노이즈가 크게 줄어 PID 제어가 안정적으로 됩니다.

8 현재 데이터 진단

현재 moserial에서 확인된 값: AX=+01471 AY=? AZ=+16120 GX≈0 GY≈0 GZ≈0

AZ = 16120 — 정상. 이상값 16384 대비 오차 264 LSB ≈ 0.016g (공장 오차 수준)
GX/GY/GZ ≈ 0 — 정상. 정지 상태에서 각속도 0에 수렴
AX = 1471 — 확인 필요. 아래 원인 중 하나
가능한 원인확인 방법해결
Bias 교정 미완료 (2분 미경과) led[2] (bias_done) 꺼져 있음 Reset 후 2분 이상 평지에서 대기
실제 5° 기울어진 상태 보드를 수평계로 확인 MPU6050 장착 각도 수정
교정 중 기울어져 있었음 Reset 후 평지에서 재교정 평지 고정 후 Reset
⚠ 빠른 교정 확인 방법:
mpu6050_ctrl.v의 READ_PERIOD_TICKS를 100으로 줄이면 교정 시간이 128초 → 약 13초로 단축됩니다.
